# General Guidelines for Code Generation

## 1. Clarity and Readability
- The generated code should be clear and easy to understand. It should follow standard coding conventions and be well-structured.
- Use meaningful variable and function names that convey their purpose.
- Include comments where necessary to explain complex logic or important decisions.

## 2. Efficiency
- The generated code should be efficient in terms of time and space complexity. Avoid unnecessary computations and optimize algorithms where possible.
- Consider the trade-offs between readability and performance, and strive for a balance that maintains clarity while improving efficiency. Include comments to explain any optimizations made that is not obvious (different from what we would normally think of in the first place).

## 3. Error Handling
- The generated code should include proper error handling to manage potential exceptions or edge cases. This may include try-catch blocks, input validation, and appropriate error messages.
- Ensure that the code fails gracefully and provides useful feedback to the user or developer when an error occurs.

## 4. Modularity and Reusability
- The generated code should be modular, with functions and classes that can be easily reused in different contexts. Avoid monolithic code that is difficult to maintain or extend.
- Encourage the use of design patterns and best practices to promote code reuse and maintainability.

## 5. Security
- The generated code should follow security best practices to protect against common vulnerabilities such as SQL injection, cross-site scripting (XSS), and buffer overflows. Validate and sanitize all user inputs, and use secure coding techniques to prevent potential security risks.
- Ensure that sensitive data is handled securely, and avoid hardcoding credentials or other sensitive information in the code.

## 6. Testing and Validation
- The generated code should include unit tests and integration tests to validate its functionality and ensure that it works as intended. Tests should cover a range of scenarios, including edge cases and potential failure points.

## 7. Git Usage
- When generating code, ensure that it is properly committed to the version control system (e.g., Git) with clear and descriptive commit messages for each small milestone. This helps maintain a clear history of changes and allows for easier collaboration with other developers.
- Follow the established branching strategy and workflow for your project, such as using feature branches for new development and pull requests for code reviews. 

# General Guidelines for Work Dispatching

## 1. Follow the Plan
- When working, follow the plan that has been outlined. This includes adhering to the specified requirements, timelines, and deliverables. If any changes are necessary, communicate them clearly and promptly.
- If you encounter any issues or obstacles that may affect the plan, report them immediately so that adjustments can be made as needed and continue the work without delay if possible.

## 2. Delegate Tasks Appropriately
- Do not hesitate to delegate tasks to team members/agents who have the appropriate skills and expertise, especially if tasks can be parallelized. This can help improve efficiency and ensure that work is completed in a timely manner.
- When delegating tasks, provide clear instructions and expectations to ensure that the work is completed correctly and meets the requirements. Delegates should also be encouraged to ask questions and seek clarification if they are unsure about any aspect of the task.

## 3. Prioritize Tasks
- Prioritize tasks based on their importance and urgency. Focus on completing high-priority tasks first, while also ensuring that lower-priority tasks are not neglected. This can help ensure that critical work is completed on time and that the overall project stays on track.
- Make sure blocking tasks are addressed as soon as possible to prevent delays in the workflow. If a task is blocked, communicate the issue to the relevant team members and work together to find a solution.
